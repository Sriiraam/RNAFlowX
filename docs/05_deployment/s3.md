# Amazon S3 Deployment

## Overview

RNAFlowX supports Amazon S3 as an optional storage backend for cloud-based RNA-seq analysis. Input datasets, reference genomes, metadata, and final outputs can be stored in Amazon S3 while computation is performed locally or using AWS Batch.

## Requirements

- AWS CLI configured (`aws configure`)
- IAM user or IAM role with S3 access
- Nextflow AWS support enabled
- Existing S3 bucket

## Example Bucket Structure

s3://rnaflowx-data/
├── raw/
├── reference/
├── metadata/
└── results/

## Running RNAFlowX with S3

```bash
nextflow run main.nf \
-profile aws,s3 \
--reads "s3://rnaflowx-data/raw/*_{1,2}.fastq.gz" \
--star_index "s3://rnaflowx-data/reference/star_index" \
--gtf "s3://rnaflowx-data/reference/gencode.v49.annotation.gtf" \
--outdir "s3://rnaflowx-data/results"
```

## Notes

- Local execution remains fully supported.
- S3 integration is optional.
- AWS Batch and S3 can be combined for scalable cloud execution.

## Troubleshooting

- Verify AWS credentials.
- Confirm IAM permissions for S3.
- Ensure bucket names and paths are correct.
- Validate configuration with:

```bash
nextflow config -profile aws,s3
```
